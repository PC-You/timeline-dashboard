Getting Started

Click the ☰ menu in the top-right and choose Load CSV to open a timestamped dataset. The tool auto-detects the timestamp column, delimiter (comma, tab, pipe, semicolon), and column structure. You can also drag and drop a file anywhere on the page.

The heatmap shows one cell per day, colored by record density. Click any cell to see that day's records in the log table below. The year sidebar on the left lets you jump between years — the label brightness indicates how much data each year contains.

Selecting Days

Beyond clicking a single day, you can build complex selections using modifier keys:

  • Ctrl+Click — Add or remove individual days from a multi-selection
  • Shift+Click — Select all cells for that day of the week (e.g. all Mondays)
  • Alt+Click — Select a full week (Sunday–Saturday)
  • W+Click — Add 7 consecutive days starting from the clicked cell (always additive)
  • M+Click — Select an entire month
  • Y+Click — Select an entire year

Hold Ctrl with any of the above to add to an existing selection instead of replacing it. Use Ctrl+A to select all, Esc to clear. Arrow keys navigate: ↑↓ move ±1 day, ←→ move ±1 week.

Focused View

When multiple days are selected, days with data appear as a collapsed list — one row per day showing the date and record count. Selections of 7 or fewer days are automatically expanded into focused view with full record tables.

On each collapsed row: click the button to peek at records in place, or click to promote the day to focused view. Focused days show a button to demote them back. This lets you investigate specific days while keeping the full selection visible.

Selection Summary

Any multi-day selection shows an aggregate Selection Summary panel above the log area. This includes total records, active vs. total days, and top values for primary/secondary columns. The summary updates live as you add or remove days from the selection.

Filtering & Highlighting

The filter bar shows faceted dropdowns for each column. Select values to narrow the dataset — excluded records disappear from the heatmap and log. Click + N more to toggle additional filter columns.

Each filter has a slide toggle in its dropdown footer (visible after selecting at least one value). The toggle switches between:

  • Filter mode (funnel icon on knob) — Excludes non-matching records
  • Highlight mode (highlighter icon on knob) — Keeps all records visible but overlays matching days with a colored radial gradient on the heatmap. The heatmap switches to grayscale so highlights stand out.

In highlight mode, a color picker appears below the dropdown with 6 preset colors. Highlighted values are also colored in the log table. Multiple highlight filters can stack — overlapping colors compound with increasing opacity. Hover the record count on collapsed multi-select rows to see a per-filter breakdown.

Notes

Add annotations at two levels:

  • Day notes — Click the pencil icon in the day header. Day notes appear as a sticky amber bar that pins to the top when scrolling through records.
  • Row notes — Click the speech-bubble icon on any record row. Row notes appear inline below the row, truncated to one line with a "Show more" toggle for longer text.

Notes are stored in the state file, not in the original data. They persist across sessions when you save and load state.

Saving Your Work

Open the ☰ menu to access:

  • Save State — Downloads a JSON file containing your notes, filter settings, highlight colors, visible columns, theme, and focused days.
  • Load State — Restores a previously saved state file. Load your CSV first, then apply the state.

You can also drag and drop CSV or JSON state files directly onto the page.

Themes

Click the gear icon in the top-right toolbar to open Settings, where you can choose from 12 color themes. The theme changes the heatmap colors and all accent elements throughout the interface. Your choice is saved in the state file.

What is this tool?

Timeline Dashboard is a read-only investigation and annotation tool for large time series datasets. It lets you visually navigate years of timestamped records through a scrollable heatmap, drill into specific days, and annotate your findings — without ever modifying the source data.

It runs entirely in your browser as a single HTML file. No server, no installation, no accounts. Load a CSV, investigate, annotate, save your state, and share it with colleagues.

What isn't this tool?

  • Not a data editor — The tool does not modify your source data. Notes and annotations live in a separate state layer.
  • Not an ETL pipeline — It consumes data but does not transform, clean, join, or export it.
  • Not a real-time dashboard — Data is loaded once per session. There is no live polling or auto-refresh.
  • Not a reporting tool (yet) — Exportable reports and charts are planned but not yet available.
  • Not a collaboration platform (yet) — State files can be shared manually. Server-based collaboration is on the roadmap.

What will this tool be?

Next up (v0.4.5)
  • Date column picker — choose the timestamp column on CSV load, with format detection (ISO, Oracle, US/EU, Unix epoch) and sample values shown per column
  • Conditional month color picker — hidden unless month indicators are enabled
  • Sample reset preserves theme — back button clears session state but keeps your palette choice
Mid-term (v0.5.x–v0.7.x)
  • Async audit — CSV parsing and aggregation off the main thread so the UI stays responsive on large datasets
  • Visible-window aggregation mode — heatmap colors scaled to what's on screen, not global, for sharper outlier detection
  • Drill-down reports exportable as PDF or HTML
  • Inline visualizations: time-of-day distribution, trend lines, day-of-week patterns
  • Comparison mode for two filter states, time ranges, or datasets
  • Large file support via chunked loading
  • Light mode and expanded theme system
Long-term (v0.7.x+)
  • Streaming data sources via WebSocket and REST APIs
  • Server-hosted version with user accounts and shared dashboards
  • Plugin system for custom metrics, columns, and integrations
  • Embeddable widget for other web applications

Guiding Principles

  • Read-only by design. The tool observes data. It does not change it.
  • Zero infrastructure. Everything works in a single HTML file. No server required.
  • Schema-driven. Any CSV with a timestamp column works with zero configuration.
  • Progressive disclosure. Simple to start, powerful when you need it.
  • Performance over polish. Fast and responsive beats beautiful and slow.
